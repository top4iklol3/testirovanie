# Приложение для отслеживания расходов на Laravel

Это простое и наглядное веб-приложение для учета личных финансов, созданное на фреймворке Laravel. Оно позволяет пользователям добавлять доходы и расходы и отслеживать свой баланс в реальном времени.

## Функционал

*   **Статистика:** В верхней части экрана всегда отображается актуальная сводка: общая сумма доходов, общая сумма расходов и текущий баланс.
*   **Добавление транзакций:** Удобная форма для быстрого добавления новой операции с указанием названия, суммы и типа (доход или расход).
*   **История операций:** Все транзакции отображаются в виде списка, отсортированного по дате добавления.
*   **Удаление транзакций:** Возможность удалять ненужные или ошибочно добавленные операции.
*   **Дизайн:** Чистый и интуитивно понятный интерфейс на Bootstrap 5, который позволяет сфокусироваться на управлении финансами.

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```sh
    git clone <адрес_вашего_репозитория>
    ```

2.  **Перейдите в папку проекта:**
    ```sh
    cd expense_schedule
    ```

3.  **Установите зависимости PHP:**
    ```sh
    composer install
    ```

4.  **Создайте файл окружения `.env`:**
    ```sh
    cp .env.example .env
    ```

5.  **Сгенерируйте ключ приложения:**
    ```sh
    php artisan key:generate
    ```

6.  **Настройте базу данных:**
    - Откройте файл `.env` и укажите ваши настройки для подключения к базе данных (например, создайте базу данных с именем `expenses`).
    - Для простоты можно использовать MySql. Для этого создайте пустой файл `database.sqlite` в папке `database/` и измените `.env` следующим образом:
      ```
      DB_CONNECTION=MySql
    
      ```

7.  **Выполните миграции для создания таблиц в БД:**
    ```sh
    php artisan migrate
    ```

8.  **Запустите локальный сервер:**
    ```sh
    php artisan serve
    ```

9.  Откройте http://127.0.0.1:8000 в вашем браузере.

## Тестирование

Проект покрыт Feature-тестами, которые проверяют все ключевые аспекты приложения.

**Ключевые сценарии, покрытые тестами:**
- Добавление транзакции (доход и расход).
- Удаление транзакции.
- Корректность расчета итогового баланса, доходов и расходов.
- Проверка валидации (нельзя создать транзакцию без названия, суммы или типа).

**Для запуска всех тестов выполните команду:**
```sh
php artisan test
```
