# Менеджер Задач (To-Do List) на Laravel

Это простое, но элегантное веб-приложение для управления задачами, созданное на фреймворке Laravel. Проект представляет собой одностраничный интерфейс для быстрого и удобного взаимодействия с задачами.

## Функционал

*   **Создание задач:** Быстрое добавление новых задач через форму.
*   **Просмотр задач:** Все задачи отображаются в виде списка.
*   **Редактирование задач:** Возможность изменять название задачи во всплывающем (модальном) окне.
*   **Удаление задач:** Удаление ненужных задач.
*   **Пометка "Выполнено":** Задачи можно отметить как выполненные или вернуть в активное состояние. Выполненные задачи визуально отличаются (перечеркнуты).

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```sh
    git clone <адрес_вашего_репозитория>
    ```

2.  **Перейдите в папку проекта:**
    ```sh
    cd to-do
    ```

3.  **Установите зависимости PHP:**
    ```sh
    composer install
    ```

4.  **Создайте файл окружения `.env`:**
    ```sh
    cp .env.example .env
    ```

5.  **Сгенерируйте ключ приложения:**
    ```sh
    php artisan key:generate
    ```

6.  **Настройте базу данных:**
    - Откройте файл `.env` и укажите ваши настройки для подключения к базе данных (например, создайте базу данных с именем `todo_list`).
    - Для простоты можно использовать SQLite. Для этого создайте пустой файл `database.sqlite` в папке `database/` и измените `.env` следующим образом:
      ```
      DB_CONNECTION=sqlite
      DB_DATABASE=/полный/путь/к/вашему/проекту/to-do/database/database.sqlite
      ```

7.  **Выполните миграции для создания таблиц в БД:**
    ```sh
    php artisan migrate
    ```

8.  **Запустите локальный сервер:**
    ```sh
    php artisan serve
    ```

9.  Откройте http://127.0.0.1:8000 в вашем браузере.

## Тестирование

Проект покрыт Feature-тестами, которые проверяют все основные операции с задачами (CRUD) и логику их завершения.

**Ключевые сценарии, покрытые тестами:**
- Отображение списка задач.
- Создание новой задачи (включая проверку на обязательное поле `title`).
- Обновление задачи (включая проверку на обязательное поле `title`).
- Удаление задачи.
- Пометка задачи как выполненной.
- Пометка выполненной задачи как невыполненной.

**Для запуска всех тестов выполните команду:**
```sh
php artisan test
```
