# Блог на Laravel

Это простой, но полнофункциональный блог, созданный на фреймворке Laravel. Проект включает в себя веб-интерфейс с аутентификацией, авторизацией и базовым дизайном на Bootstrap.

## Функционал

*   **Аутентификация и регистрация:** Пользователи могут создавать аккаунты, входить и выходить из системы.
*   **Управление постами:** Авторизованные пользователи могут создавать, редактировать и удалять посты.
*   **Авторизация (Политики безопасности):** Пользователи могут редактировать и удалять **только свои** посты.
*   **Комментирование:** Авторизованные пользователи могут оставлять комментарии к постам.
*   **Категории:** Посты можно относить к одной или нескольким категориям.
*   **Фильтрация и Поиск:** Реализована фильтрация постов по категориям и текстовый поиск по заголовкам и содержимому.
*   **Дизайн:** Простой и чистый адаптивный дизайн на Bootstrap 5.

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```sh
    git clone <адрес_вашего_репозитория>
    ```

2.  **Перейдите в папку проекта:**
    ```sh
    cd blog
    ```

3.  **Установите зависимости PHP:**
    ```sh
    composer install
    ```

4.  **Создайте файл окружения `.env`:**
    ```sh
    cp .env.example .env
    ```

5.  **Сгенерируйте ключ приложения:**
    ```sh
    php artisan key:generate
    ```

6.  **Настройте базу данных MySQL:**
    - Убедитесь, что у вас запущен сервер MySQL.
    - Создайте новую базу данных для проекта (например, `blog`).
    - Откройте файл `.env` и укажите ваши настройки для подключения:
      ```
      DB_CONNECTION=mysql
      DB_HOST=127.0.0.1
      DB_PORT=3306
      DB_DATABASE=blog
      DB_USERNAME=root
      DB_PASSWORD=
      ```

7.  **Импортируйте дамп базы данных:**
    В репозитории находится файл `database/blog.sql` . Импортируйте его в созданную вами базу данных с помощью команды:
    ```sh
    mysql -u [имя_пользователя] -p [имя_базы_данных] < database/dump.sql
    ```
    *(Замените `[имя_пользователя]` и `[имя_базы_данных]` на ваши значения)*.
    
    **Примечание:** Этот шаг создает все необходимые таблицы и заполняет их начальными данными (категориями), поэтому команды `php artisan migrate` и `php artisan db:seed` выполнять не нужно.

8.  **Запустите локальный сервер:**
    ```sh
    php artisan serve
    ```

9.  Откройте http://127.0.0.1:8000 в вашем браузере.

## Тестирование

Проект покрыт Feature-тестами, которые проверяют основной функционал безопасности и управления постами.

**Ключевые сценарии, покрытые тестами:**
- Гость не может создать пост.
- Авторизованный пользователь может создать пост.
- Пользователь не может редактировать или удалять чужие посты.
- Пользователь может редактировать и удалять свои посты.

**Для запуска всех тестов выполните команду:**
```sh
php artisan test
```
